# Story 001: App Shell and Routing

Status: planned
Created: 2026-05-19T10:45:00Z

## User Story

As a visitor, I want a clear app structure with predictable navigation, so that I can browse stays, open details, and return to saved favorites.

## Acceptance Criteria

- Given the app loads, when the visitor arrives, then the visitor sees the working discovery experience as the primary screen.
- Given the visitor navigates between browse, detail, and favorites routes, when navigation completes, then the app keeps a coherent layout and page title context.
- Given the visitor uses keyboard navigation, when focus moves through the shell, then focus order is logical and visible.

## Notes

Do not build a marketing landing page as the first screen. The first screen should be the usable product experience.
