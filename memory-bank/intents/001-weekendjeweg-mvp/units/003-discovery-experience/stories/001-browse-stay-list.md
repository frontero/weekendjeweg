# Story 001: Browse Stay List

Status: planned
Created: 2026-05-19T10:45:00Z

## User Story

As a visitor, I want to scan weekend stays in a clear list, so that I can quickly decide which options are worth opening.

## Acceptance Criteria

- Given the catalog contains stays, when the browse route loads, then stay cards are displayed.
- Given each stay card renders, when the visitor scans it, then name, location, price label, highlights, and tags are visible.
- Given the visitor uses a mobile viewport, when the list renders, then content remains readable and controls do not overlap.

## Notes

Cards may be used for repeated stay items. Page sections should stay unframed and scannable.
